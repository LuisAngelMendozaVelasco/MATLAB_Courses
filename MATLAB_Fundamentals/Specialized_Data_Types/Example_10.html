Task 1

By default, the possible values and the names of the categories will be determined automatically from the data. You can specify these as additional inputs to categorical, where the second input indicates the unique category values in the original array, and the third input indicates the names that correspond to these categories.
v = [ 10 5 0 0 ];
levels = ["beg" "mid" "last"];
categorical(v,[0 5 10],levels)
ans =
last mid beg beg
y = categorical(x, 1:4, xlevels)
y = 1×9 categorical
small huge tiny tiny small small tiny small huge

Task 2

If your categories have an inherent ordering – for example, “low”, “medium”, and “high” – you can specify this with an optional property "Ordinal":
v = [ 10 5 0 0 ];
levels = ["beg" "mid" "last"];
c = categorical(v,...
[0 5 10],levels,...
"Ordinal",true)
c > "mid"
ans =
1 0 0 0
This allows you to compare the values using operators such as greater than (>) and less than (<).
y = categorical(x, 1:4, xlevels, "Ordinal", true)
y = 1×9 categorical
small huge tiny tiny small small tiny small huge

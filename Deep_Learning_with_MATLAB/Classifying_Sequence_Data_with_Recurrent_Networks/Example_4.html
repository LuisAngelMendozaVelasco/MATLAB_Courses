Task 1

During training, the network learns to associate the training signals and labels. The network may have a high training accuracy, but a network is only useful if it can generalize to new signals. You should use a separate validation and test data set to evaluate if the network can classify signals it has not yet seen.
You can split the data set by using splitlabels.
idx = splitlabels(labels,[0.9,0.05])
The first input is the labels of the entire data set. You can specify ratios between the different sets in a vector. In this example, 95% of the signals will go in the training data set, 5% will go in the validation data set, and the rest (5%) will go in the testing data set.
idx = splitlabels(labels, [0.8 0.1])

Task 2

The indices are stored in a cell array where each element contains the training, validation, and testing indices, in that order. For example, you can get the training indices with idx{1}.
Once you extract the indices, you can use them to partition the data set using indexing. sigdata and labels are aligned because the folders2labels function reads files in the same order as signalDatastore. To keep this alignment, use the same indices to partition sigdata and labels.
trainidx = idx{1};
validx = idx{2};
testidx = idx{3};
